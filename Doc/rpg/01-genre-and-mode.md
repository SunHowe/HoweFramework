# 01 · RPG 子类型与玩法模式

> 本章解决"我做的 RPG 是什么类型"的问题。**先定类型,后定系统**。

---

## 一、RPG 子类型分类

### 1.1 四大主类

| 子类型 | 战斗节奏 | 典型代表 | 玩家输入频率 |
|--------|----------|---------|--------------|
| **回合制 RPG**(Turn-Based RPG) | 离散回合,按顺序出手 | 梦幻西游、最终幻想系列、仙剑奇侠传 | 每回合 1 次决策(半自动)或 0 次(全自动)|
| **SRPG**(策略 RPG / 战棋) | 离散回合 + 格子 / 战术 | 火焰纹章、皇家骑士团、梦幻模拟战 | 每回合 N 次决策,每个己方单位 1 次 |
| **ARPG**(动作 RPG) | 实时连续 | 暗黑破坏神、Path of Exile、原神 | 实时高频输入 + 技能冷却 |
| **MMORPG**(大型多人在线 RPG) | 实时 + 多人同步 | 魔兽世界、FF14、原神(开放世界部分)| 实时输入 + 网络同步 |

### 1.2 混合类型

| 类型 | 描述 | 代表 |
|------|------|------|
| **半即时回合制** | 行动条累积 + 阈值出手 | 轨迹系列、女神异闻录 |
| **ATB 回合制** | Active Time Battle(有"等待槽"概念)| FF4-9 |
| **即时回合制** | 时间暂停选指令,执行时实时 | 极乐迪斯科、时光之帽 |
| **卡牌 RPG** | 抽牌 + 组牌 + 释放 | 杀戮尖塔、月圆之夜 |
| **Roguelike RPG** | 回合制 + 永久死亡 + 随机生成 | 哈迪斯、不思议的皇冠 |
| **放置 RPG** | 离线战斗 + 收集 + 养成 | 剑与远征、最强祖师 |

### 1.3 选哪个?—— 决策树

```
我做的 RPG 主要体验是什么?
    │
    ├─ 我要"下指令 + 看动画"的慢节奏 → 回合制 RPG
    │      │
    │      └─ 玩家要"自己掌控节奏"还是"看 AI 自动打"?
    │              │
    │              ├─ 玩家掌控(每回合选技能 / 目标 / 物品)
    │              │       → 半自动模式(30s 指令窗口 + 超时自动判定)
    │              │
    │              └─ AI 自动(每回合只看动画)
    │                      → 全自动模式
    │
    ├─ 我要"战术 + 站位 + 角色培养" → SRPG(战棋)
    │
    ├─ 我要"刷怪 + 装备 + Build" → ARPG(动作)
    │
    └─ 我要"在线 + 多人 + 长期养成" → MMORPG
```

## 二、半自动 vs 全自动 —— 核心差异

### 2.1 全自动模式

**特点**:玩家只选"出招 / 被打",不选技能 / 目标 / 物品。

**典型**:
- 神仙道、QQ 水浒、刀塔传奇(简化模式)

**优势**:
- 上手简单
- 适合"看动画就好"的重度剧情
- 适合碎片时间(地铁上挂机)

**劣势**:
- 没有"决策感",玩家是观众不是参与者
- 同质化严重(全靠数值碾压)

### 2.2 半自动模式

**特点**:每回合开始前玩家有 N 秒下指令,下完后看动画执行。

**典型**:
- 梦幻西游、大话西游、问道、口袋妖怪

**核心机制**:
- 指令窗口(常见 30s)
- 超时自动判定(常见 = 默认物理攻击)
- 收指令 + 锁定 + 不可改

**优势**:
- 决策密度高(每个回合都要思考)
- "半自动" 的紧张感(被时钟追)
- 战术组合空间大(技能 / 目标 / 物品 / 召唤 / 保护 …)

**劣势**:
- 上手难
- 实时反馈慢(看完一整个回合才能知道结果)
- 网络延迟 / 掉线处理复杂

### 2.3 半自动的"超时自动判定"是关键设计

**默认操作**(梦幻西游 / 大话西游 / 问道 式):
- 超时 = 自动物理攻击(无目标时随机)
- 不是"跳过该单位" —— 这样会破坏节奏

**变体**:
- 超时 = 防御(降低风险)
- 超时 = 自动上次指令(降低学习成本)
- 超时 = 全队自动(常见,但削弱玩家决策)

**设计原则**:**默认操作要"有意义但不最优"** —— 让玩家觉得"反正输了也比这好"。

## 三、回合制 RPG 的核心体验 = "决策密度"

### 3.1 什么是决策密度

单位时间内玩家做的决策数 × 每个决策的影响度。

| 类型 | 决策密度 | 体验 |
|------|---------|------|
| 全自动 | 极低(0-1 次/回合)| 看动画 |
| 半自动 | 中等(每回合 N 次决策)| 紧张 + 战术 |
| SRPG | 高(每回合 N × M 单位决策)| 深度策略 |
| ARPG | 高(每秒 N 次决策)| 实时反应 |

### 3.2 半自动的"半"在哪

- "半"在**指令窗口**(30s 是最常见)
- "半"在**超时自动判定**(没下完指令自动补默认)
- "半"在**玩家决策权** vs **AI 自动战斗** 的平衡

### 3.3 决策密度的设计杠杆

| 杠杆 | 加大决策密度 | 减小决策密度 |
|------|--------------|--------------|
| 指令窗口 | 15s(紧迫)| 60s(宽松)|
| 指令数 | 每单位多指令(技能 + 物品 + 召唤 + 保护)| 单指令(技能)|
| 单位数 | 5 人队 + 5 宠(10 单位)| 1 人队(单机 RPG)|
| 状态数 | 多 buff / debuff 需要判断 | 无状态 |
| 战术空间 | 站位 / 阵法 / 装备词条 | 纯数值 |

## 四、回合制 vs ARPG 的设计差异

### 4.1 战斗节奏

| 维度 | 回合制 | ARPG |
|------|--------|------|
| 单位时间行动数 | 1-2 次/秒(所有单位)| 10-50 次/秒(单玩家)|
| 玩家反馈延迟 | 高(看完一整个回合)| 极低(实时 hit feedback)|
| 数值感 | 强(每次伤害数字大)| 中(每次伤害数字小但频次高)|
| 策略感 | 强(回合决策)| 弱(操作熟练度)|
| 沉浸感 | 弱(回合外思考)| 强(实时战斗)|
| 美术资源 | 静态立绘 + 特效(成本低)| 动作捕捉 + 实时动作(成本高)|

### 4.2 系统差异

| 系统 | 回合制 | ARPG |
|------|--------|------|
| 战斗循环 | FSM(回合 → 行动 → 回合结束)| 实时 tick + 帧同步 |
| 出手顺序 | 按速度排序 | 实时(无排序)|
| 持续效果(Tick) | 回合开始 / 回合结束触发 | 每 N 帧触发 |
| 倒计时 | `GameTimerManager`(绝对时间)| `MonoBehaviour.Update` 或帧计时器 |
| 死亡 | 倒地 / 死亡 / 鬼魂 | 真死 + 复活机制 |
| 阵营 / 站位 | 站位(阵法 / 阵位)| 无(自由移动)|
| 召唤兽 | 1 宠(参战)| 多宠物(召唤 + 切宠)|
| 物品 | 回合内有限使用 | 冷却 / 数量 |
| AI | 回合决策(选技能 + 选目标)| 实时决策(BT + 状态机)|

### 4.3 选择哪个 —— 决策

```
我做的 RPG 主要体验是什么?
    │
    ├─ 慢节奏 + 强决策 + 弱反应 → 回合制
    ├─ 快节奏 + 弱决策 + 强反应 → ARPG
    └─ 慢节奏 + 强决策 + 强反应 → 半即时 / ATB / 即时回合制
```

## 五、回合制 RPG 的"模式"细分

### 5.1 纯回合制 vs ATB

| 维度 | 纯回合制(梦幻西游式) | ATB(FF4-9 式)|
|------|--------------------|--------------|
| 行动条 | 整回合按速度排 | 实时累积到阈值 |
| 时间感 | "回合" | "时间槽"|
| 玩家输入时机 | 回合开始前 | 行动条满了就出手 |
| 实时反应要求 | 无 | 有(槽快满了要选)|
| 体验关键词 | 思考 + 等待 | 反应 + 紧张 |

### 5.2 单人 vs 队伍

| 维度 | 单人 | 队伍 |
|------|------|------|
| 典型 | 仙剑、FF | 梦幻西游、口袋妖怪 |
| 决策数 | 每回合 1 次 | 每回合 N 次(N = 队伍人数)|
| 队伍平衡 | 不需要(单人)| 关键(队伍中各职业互补)|
| 站位 | 不需要 | 站位 / 阵法 |

### 5.3 同屏 vs 切换(2D / 2.5D / 3D)

| 维度 | 同屏 2D | 同屏 2.5D | 切换 3D(战斗画面独立)|
|------|----------|------------|---------------------|
| 战斗画面 | 跟世界同屏 | 半 3D | 切到独立战斗场景 |
| 美术成本 | 低 | 中 | 高 |
| 战前预期 | 低(明雷可见怪)| 中 | 高(切场景是"开打"信号)|
| 典型 | 早期 RPG | 史莱姆牧场 | FF、梦幻西游 |

## 六、SRPG(策略 RPG / 战棋)的差异

### 6.1 核心差异

| 维度 | 回合制 RPG | SRPG |
|------|------------|------|
| 战场 | 无 / 抽象 | 棋盘格(6×6 / 8×8 / 网格)|
| 单位数 | 5-10 人队 | 10-50 个单位 |
| 站位 | 阵法(属性加成)| 格子(战术意义)|
| 移动 | 无 | 每回合移动 + 攻击(分开消耗)|
| 地形 | 无 | 影响(树林 +30% 躲避 / 山地 +50% 防御)|
| 决策维度 | 技能 / 目标 / 物品 | + 移动 + 站位 + 地形 |

### 6.2 SRPG 的关键系统

- **格子系统**(6×6 / 8×8 / 网格 / 六角格)
- **移动力**(MP,移动点,不是字面"MP")
- **攻击范围**(近战 1 格 / 远程 2-3 格 / 直线 / 扇形)
- **反击**(被近战时自动反击)
- **地形加成**(树林、山地、河川、城墙)
- **站位职业差异化**(前排坦克 / 中排输出 / 后排辅助)

---

## 七、MMORPG 的差异

### 7.1 核心差异

| 维度 | 单机 / 弱联机 RPG | MMORPG |
|------|--------------------|---------|
| 同步 | 弱(本地)| 强(网络)|
| 服务器权威 | 无 | 有(防作弊)|
| 玩法 | 剧情 + 战斗 | 社交 + 副本 + 战斗 |
| 数值 | 故事驱动 | 经济系统驱动 |
| 战斗 | 半自动 / 全自动 | 实时(强制输入)或 ATB |

### 7.2 MMORPG 战斗的特殊设计

- **服务器权威**(防外挂) —— 客户端只发指令,服务端裁决
- **状态同步**(每秒 N 次状态广播)
- **属性压缩**(显示数值要符合玩家心理预期,不能差太多)
- **战斗节奏**(多人副本有"坦克拉住仇恨 + DPS 输出 + 治疗保人"的铁三角)

## 八、决策:开始一个 RPG 项目时该选哪种?

### 8.1 决策树

```
我的项目预算 / 目标用户 / 核心体验?
    │
    ├─ 单人 / 小团队 / 中低预算 / 慢节奏策略
    │      └─ 回合制 RPG + 半自动模式(梦幻西游式)
    │
    ├─ 单人 / 大预算 / 实时反应
    │      └─ ARPG(暗黑 / 原神 / 鬼泣风)
    │
    ├─ 多人 / 服务器 / 长期运营
    │      └─ MMORPG(实时战斗 + 网络同步)
    │
    └─ 单人 / 中预算 / 强策略 / 棋盘
           └─ SRPG(战棋)
```

### 8.2 一句话总结

| 项目类型 | 推荐 |
|---------|------|
| 单人 / 小团队 / 中预算 | **回合制 RPG + 半自动**(梦幻西游式) |
| 单人 / 中大预算 / 视觉优先 | ARPG |
| 长期运营 / 多人 | MMORPG |
| 单人 / 强策略 / 小体量 | SRPG |

## 九、本章沉淀的设计模式

沉淀的不是具体某款 RPG,而是**所有 RPG 项目共用的设计模式**:

1. **半自动 vs 全自动模式**:决定战斗节奏和决策密度。
2. **回合制 vs 实时**:决定 FSM 还是实时 tick。
3. **单机 vs 多人**:决定是否需要服务端权威。
4. **同屏 vs 切换**:决定美术成本和战斗体验。
5. **超时自动判定**:半自动的关键设计,默认操作要"有意义但不最优"。
6. **决策密度**:决定"半"的程度。
7. **指令窗口时长**(15s-60s):决定紧张感。
8. **倒地 ≠ 死亡 ≠ 鬼魂**:处理 HP=0 的三种边界。

具体数值(30s 指令窗口 / 150 回合上限 / 5 人队 + 1 宠 vs 14 怪)是梦幻西游的数值,**不**沉淀到这里。
