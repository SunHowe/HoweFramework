# 04 · 技能系统

> 本章沉淀回合制 RPG 的技能系统设计 —— 不含具体游戏数值,只沉淀"为什么这样设计"。

---

## 一、技能系统的本质

### 1.1 技能是"战斗决策的最小单位"

- 每个技能 = 一个**指令**(玩家和 AI 的最小决策单位)。
- 每个技能 = 一段**效果**(伤害 / 治疗 / 状态 / 召唤 / 保护 / 防御)。
- 每个技能 = 一组**约束**(MP 消耗 / CD / 目标类型 / 职业限制)。

### 1.2 技能系统的设计目标

1. **可配置** —— 策划改 `.json` 改技能,不改代码。
2. **可扩展** —— 新增技能不影响现有数据。
3. **可分类** —— 玩家能快速理解技能定位。
4. **可平衡** —— 数值 / 效果可独立调整。

## 二、6 类技能分类(回合制 RPG 标准)

### 2.1 标准分类

| 类别 | 描述 | 主要使用者 | 典型技能 |
|------|------|-----------|---------|
| **物理** | 用物伤造成物理伤害 | 战士 / 刺客 / 弓手 | 横扫千军、穿刺、连击 |
| **法术** | 用法伤造成法术伤害 | 法师 / 道士 | 龙卷雨击、火球、冰锥 |
| **封印** | 限制对方行动 | 控制职业 | 失心符、定身符、睡眠 |
| **治疗** | 恢复己方 HP | 辅助职业 | 推气过宫、加血、群体治疗 |
| **防御** | 本回合不出手 / 减伤 | 所有职业 | 防御、护盾 |
| **复活** | 救起倒地队友 | 辅助职业 | 复活、起死回生 |

### 2.2 为什么是这 6 类,不是 4 类或 8 类?

**设计意图**:这 6 类是"决策维度的最小完备集合"。

少了"封印" → 失去控制职业的差异化
少了"治疗" → 辅助职业跟输出职业混
少了"防御" → 玩家不会"等"
多了"召唤兽" / "特技" / "陷阱" 等 → 这些是**子分类**,不是"独立类别"

**结论**:6 类是"决策维度足够,职业定位清晰"的平衡点。

### 2.3 子分类(常见变体)

| 子分类 | 描述 | 属于哪一类 |
|--------|------|----------|
| 群体物理 | 横扫千军(打 3 个目标)| 物理 |
| 多段攻击 | 连击(同一目标打 N 次)| 物理 |
| 持续伤害 | 中毒符、灼烧 | 法术 |
| 召唤兽 | 召唤一只宝宝参战 | 独立系统(见 `06-pet-and-formation.md`)|
| 特技 | 消耗愤怒的大招 | 独立系统(见 `07-anger-and-items.md`)|
| 道具 | 药品 / 武器 | 独立系统(见 `07-anger-and-items.md`)|
| 逃跑 | 离开战斗 | 系统操作 |
| 捕捉 | 收服怪物为召唤兽 | 召唤兽系统 |
| 快捷施法 | 上一次法术的快速按钮 | UI 优化 |

## 三、技能数据结构

### 3.1 标准字段

```csharp
public class SkillData {
    public int SkillId;          // 唯一 id
    public string Name;          // 名称
    public SkillType Type;       // 6 类之一
    public TargetType TargetType;// 目标选择规则
    public int MpCost;           // MP 消耗
    public int CdRound;          // CD 回合数(0 = 无 CD)
    public int EffectId;         // 效果 id(对应 SkillEffect ScriptableObject)
    public string Formula;       // 伤害公式字符串(ExpressionManager 解析)
    public int[] ApplyStates;    // 施加的状态 stateIds
    public int BaseLevel;        // 最低使用等级
    public JobType JobLimit;     // 职业限制(可选)
}
```

### 3.2 目标选择规则

| TargetType | 描述 | UI |
|------------|------|-----|
| `Self` | 自己 | 不需要选 |
| `SingleAlly` | 单个队友 | 选 1 个己方单位 |
| `AllAlly` | 所有队友 | 不需要选 |
| `SingleEnemy` | 单个敌人 | 选 1 个敌方单位 |
| `AllEnemy` | 所有敌人 | 不需要选 |
| `SingleAny` | 单个任意单位 | 选 1 个单位 |
| `AllAny` | 所有单位 | 不需要选 |
| `DeadAlly` | 倒地队友(复活用)| 选 1 个倒地己方单位 |

### 3.3 公式字符串(`ExpressionManager`)

```csharp
// 示例 1:基础物理伤害
"max(1, $atk - $def)"

// 示例 2:含暴击
"max(1, $atk - $def) * (1 + $isCrit * 0.5)"

// 示例 3:含波动
"max(1, $atk - $def) * (0.95 + $randomNext() * 0.1)"

// 示例 4:多段攻击(三刀)
"max(1, ($atk + $lv) * 1.0) + max(1, ($atk + $lv) * 1.1) + max(1, ($atk + $lv) * 1.2)"

// 示例 5:治疗
"($lv + $cultivation) * 4 + $healPower * 0.6"
```

**优势**:
- 策划改公式不用改代码。
- 可注册任意 token(`$atk` / `$def` / `$lv` / `$cultivation` / `$healPower`)。
- `userData` 传 caster + target 上下文。

## 四、技能释放的三步流程

### 4.1 标准流程

```
1. 选技能(玩家下指令时)
2. 选目标(根据 TargetType 决定)
3. 执行(结算 MP / CD / 伤害 / 状态 / 胜负)
```

### 4.2 释放的"结算步骤"

```
1. 检查目标是否有效(死亡 / 隐身 / 无敌)
2. 检查 MP 够不够(不够 → 失败提示)
3. 检查 CD(冷却中 → 失败提示)
4. 扣 MP(Cost 安全扣)
5. 进入 CD
6. 计算效果(读公式字符串)
7. 应用伤害 / 治疗 / 状态 / 召唤 / 复活
8. 播动画 + 弹伤害数字
9. 广播"技能释放"事件(给 UI / 录像)
```

### 4.3 异常处理

| 异常 | 处理 |
|------|------|
| MP 不足 | 提示"魔法不足",技能不进入 CD |
| 目标已死 | 提示"目标已死亡",技能不进入 CD |
| 目标隐身 | 提示"目标隐身",技能不进入 CD |
| 目标无敌 | 提示"目标无敌",技能不进入 CD |
| 施法者被封 | 提示"已被封印",技能不进入 CD |

### 4.4 技能状态(玩家 vs 怪物)

| 玩家施法 | AI 施法 |
|---------|--------|
| 选技能 + 选目标(玩家输入)| AI 行为树选技能 + 选目标 |
| MP 不足时提示玩家 | AI 看 MP 不足自动换技能 |
| 锁定(下完不可改)| 自动执行 |

## 五、伤害公式的设计原则

### 5.1 伤害公式的三个维度

| 维度 | 常见模式 | 设计意图 |
|------|---------|---------|
| **基础计算** | `max(1, 攻击 - 防御)` | 简单 |
| **暴击 / 抗暴** | `damage * (1 + isCrit * 暴击倍数 - 抗暴 * 削减)` | 数值膨胀 |
| **波动 / 随机** | `damage * (0.95 + random * 0.1)` | 不确定感 |
| **属性加成** | `damage * (1 + 法伤 / 100)` | 装备 / buff 加成 |
| **抗性** | `damage * (1 - 法抗 / 100)` | 多维防御 |
| **穿透** | `damage + 穿透值` | 忽视抗性 |
| **护甲** | `damage = max(1, damage - 护甲)` | 固定减伤 |

### 5.2 公式的常见误用

| 误用 | 问题 |
|------|------|
| **基础攻击 = 0 时伤害也 = 0** | 永远有"最小伤害 1",保证玩家有反馈 |
| **百分比叠加无限** | 后期数值膨胀,平衡失效 |
| **公式不透明** | 玩家看不到公式,不知道"为什么这伤害" |
| **公式被策划改崩** | 公式字符串要做"语法 + 范围"校验 |

### 5.3 公式应该可配置 vs 不可配置

| 可配置 | 不可配置 |
|--------|----------|
| 数值(常数 / 系数)| 公式逻辑结构 |
| 公式中的 token 含义 | 公式的"语法"(必须走 ExpressionManager)|
| 公式的输入(谁传 $atk / $def)| 公式的"输出"(最终伤害的物理含义)|

## 六、技能系统的常见边界

### 6.1 职业限制

部分技能只能特定职业用:
- "横扫千军" → 战士系
- "失心符" → 道士系
- "推气过宫" → 僧侣系

**实现**:`JobLimit` 字段 + 检查当前角色职业。

### 6.2 等级限制

部分技能需要达到一定等级才能学 / 用:
- "横扫千军 Lv1" → 等级 ≥ 40
- "失心符 Lv5" → 等级 ≥ 80 + 技能等级 ≥ 5

**实现**:`BaseLevel` 字段 + 检查当前等级。

### 6.3 装备限制

部分技能需要装备对应武器:
- "弓系技能" → 必须装备弓
- "法系技能" → 不能装备盾

**实现**:`WeaponType` 字段 + 检查当前武器。

### 6.4 技能升级

| 升级类型 | 描述 |
|---------|------|
| **技能等级** | 同一技能有 Lv1-9,每级提升效果 |
| **技能精通** | 装备 / 状态给技能加等级加成 |
| **技能强化** | 特定技能有"进阶版"(基础技能 + 后缀)|

### 6.5 技能学习

- 技能 **可以学**(玩家主动学)
- 技能 **可以忘**(遗忘已学技能)
- 技能 **可以升级**(用技能点 / 自动升级)

## 七、技能系统的常见反模式

### 7.1 单一最强技能

**问题**:某个技能太强,所有玩家都用它。

**修正**:
- 限制使用次数(CD / 次数限制)
- 削弱效果(数值 / 倍率)
- 增加"免疫 / 抗性"机制(对方能克制)
- 加入"互克表"(火克风 / 风克土 / …)

### 7.2 技能同质化

**问题**:不同技能效果相同,玩家选择无意义。

**修正**:
- 不同技能有不同决策维度(单体 / 群体 / MP / CD / 目标)
- 引入"职业特色"(物理职业的物理技能,法系职业的法系技能)
- 引入"协同"(技能 A 增强技能 B)

### 7.3 技能膨胀

**问题**:技能越加越多,玩家记不住。

**修正**:
- 技能分组(基础 / 进阶 / 终极)
- 技能"自动"按钮(常用技能可标记"自动施法")
- 技能"快捷栏"(只显示最常用的 6-12 个技能)

## 八、技能系统的实现要点

### 8.1 技能数据存储

| 存储 | 适用 |
|------|------|
| **ScriptableObject** | 静态技能(策划在编辑器配)|
| **DataTable(Luban)** | 大型技能表(> 100 个技能)|
| **JSON** | 灵活配置(玩家自制技能 / DLC 技能)|

### 8.2 技能表的索引

- 按 `SkillId` 索引(`Dictionary<int, SkillData>`)
- 按职业索引(`Dictionary<JobType, List<SkillId>>`)
- 按类型索引(`Dictionary<SkillType, List<SkillId>>`)

### 8.3 技能释放的事件

- `OnSkillCast(caster, target, skill, result)` —— 广播"技能释放"事件
- UI 订阅 → 播动画
- 录像订阅 → 存序列
- AI 订阅 → 学习玩家模式(高级)

### 8.4 技能 AI 决策

- 简单 AI:if MP > 80% 用最强技能,else 用普通攻击
- 进阶 AI:行为树(HP < 30% → 加血;否则单体技能 → 群体技能 → 普攻)
- 高级 AI:玩家模式学习(模仿玩家常用技能)

## 九、本章沉淀的设计模式

沉淀的不是具体某款 RPG 的技能数值,**而是技能系统的设计骨架**:

1. **6 类技能分类** —— 物理 / 法术 / 封印 / 治疗 / 防御 / 复活(决策维度最小完备集合)。
2. **技能数据结构** —— id / 名称 / 类型 / 目标 / MP / CD / 效果 / 公式 / 状态 / 等级 / 职业。
3. **目标选择 8 种** —— Self / SingleAlly / AllAlly / SingleEnemy / AllEnemy / SingleAny / AllAny / DeadAlly。
4. **公式字符串** —— `ExpressionManager` 解析,策划可配置。
5. **释放三步** —— 选技能 / 选目标 / 结算。
6. **结算步骤** —— 检查 → 扣 MP → CD → 计算效果 → 应用 → 播动画 → 广播。
7. **伤害公式三维度** —— 基础 + 暴击抗暴 + 波动 / 抗性 / 穿透。
8. **常见反模式** —— 单一最强技能 / 技能同质化 / 技能膨胀。

具体数值(横扫三刀衰减系数、推气过宫 4 回合持续、失心符基础命中率 55%)是梦幻西游的数值,**不**沉淀到这里。
