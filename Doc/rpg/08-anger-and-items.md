# 08 · 愤怒、特技与物品

> 本章沉淀回合制 RPG 的愤怒 / 特技 / 药品系统设计 —— 不含具体游戏数值,只沉淀"为什么这样设计"。
>
> **架构澄清**:本章涉及的所有"中毒 / 灼烧 / 隐身 / 加血"等持续效果,都是 **Provider**(典型 = buff)。详见 [`06-buff-system.md`](06-buff-system.md) 和 [`05-state-component.md`](05-state-component.md)。

## 一、愤怒系统

### 1.1 愤怒的本质

愤怒 = "战斗内累积的特殊资源":
- 进入战斗携带一部分(基础 + 临时符)
- 战斗中累积(受到伤害 / 治疗 / 道具)
- 消耗(释放特技)
- 愤怒清零(死亡时)

### 1.2 愤怒的设计目标

1. **大特技需要"长期累积"** —— 不能每个回合放,需要累积 N 个回合。
2. **反击 / 防守的"奖励"** —— 被打越多,愤怒越多,反杀机会越大。
3. **翻盘可能** —— 残血一波反杀(愤怒满了放大特技)。

### 1.3 愤怒的"加怒"来源

| 来源 | 描述 |
|------|------|
| **受到伤害** | 按损失气血占总血量的比例分段加(主要来源) |
| **受到治疗** | 满血也加(部分游戏)|
| **道具 / 酒** | 蛇胆酒 / 珍露酒(直接加 N 点)|
| **法宝** | 乾坤玄火塔(每回合按比例回)|
| **技能** | 部分技能可以加怒(罕见) |

### 1.4 愤怒的"减怒"来源

| 来源 | 描述 |
|------|------|
| **释放特技** | 直接消耗 |
| **死亡** | 清 0 |
| **被救起** | 救起也是 0 |
| **技能减怒** | 笑里藏刀 / 西天梵音(减对方 N 点)|

### 1.5 愤怒的关键设计模式:分段加怒

**反例**(线性加怒):
- 被打 1% 血 = 加 1 点愤怒
- 被打 99% 血 = 加 99 点愤怒
- 不平衡(玩家 1 回合被打 99% 血很常见,但愤怒只加 99)

**正解**(分段加怒):
- 损失 0-3% → 加 1 点
- 损失 3-10% → 加 3 点
- 损失 10-20% → 加 10 点
- 损失 20-30% → 加 15 点
- 损失 30-50% → 加 25 点
- 损失 50-80% → 加 40 点
- 损失 80-99% → 加 55 点

**设计意图**:
- 残血受击愤怒累积更快
- 翻盘可能(残血一波反杀)
- 满血受击愤怒累积慢(避免"满血放特技")

### 1.6 愤怒的"上限"

| 上限 | 描述 |
|------|------|
| **常见上限** | 150(梦幻西游 / 大话西游 / 问道) |
| **装备上限** | 临时符 / 愤怒腰带 |
| **战斗上限** | 进入战斗时已设上限 |

### 1.7 愤怒腰带

愤怒腰带 = 装备,特技消耗 × 0.8(削 20%)。
- 例:罗汉金钟消耗 150 愤怒 → 带愤怒腰带时消耗 120 愤怒

### 1.8 愤怒的"主动技能消耗血不增自身怒"

**重要规则**:主动技能消耗血(如横扫千军反伤 7-13%)**不增自身怒**。
- 设计意图:防止"主动消耗血加愤怒"的套利。

### 1.9 多段攻击的愤怒

**重要规则**:横扫千军 / 连环击等多段攻击**按每段分别计算**愤怒。
- 例:横扫千军 3 刀,每刀按"被打的伤害比例"分段加怒
- 设计意图:多段攻击 = 多次愤怒累积 = 残血反杀的"放大器"

### 1.10 愤怒数据结构

```csharp

    public int MaxAnger;        // 上限(常见 150)
    public int CurrentAnger;    // 当前值
    public float CarryRatio;    // 进入战斗携带比例(常见 50%)
    public int TempBonus;       // 临时符加成
}

public sealed class AngerComponent : GameComponentBase {
    public AngerData Data;
    
    // 受击加怒(分段)
    public void AddAngerFromDamage(long damage, long maxHp, bool isMultiHit) {
        // 多段攻击:每段分别计算
        // 单次攻击:按比例分段
    }
    
    // 释放特技消耗

        if (Data.CurrentAnger < amount) return false;
```

## 二、特技系统

### 2.1 特技的本质

特技 = "消耗愤怒释放的大招":
- 释放成本高(愤怒值)
- 效果强(全体 buff / 全体伤害 / 复活)
- 不可频繁使用(愤怒需要累积)

### 2.2 特技的设计目标

1. **翻盘可能** —— 残血愤怒满了放大特技,逆转战局。
2. **决策维度** —— 愤怒有限,选择何时放哪个特技。
3. **职业差异化** —— 不同职业有不同的特技(辅助 / 输出 / 控制)。

### 2.3 特技的常见分类

| 类别 | 描述 | 典型 |
|------|------|------|
| **群体 buff** | 己方全体获增益 | 罗汉金钟(全体法防 +50%) |
| **群体 debuff** | 敌方全体获减益 | 失魂幡(全体减治疗效果) |
| **群体伤害** | 敌方全体受伤害 | 飞沙走石(全体法术伤害)|
| **群体复活** | 己方全体倒地复活 | 普陀山的"灵动九天"|
| **单体大治疗** | 单个目标大量治疗 | 推气过宫(强化版)|
| **单体大伤害** | 单个目标大量伤害 | 泰山压顶 |
| **解除异常** | 净化己方 | 观音赐福(净化 + 治疗)|
| **特殊** | 非常规效果 | 乾坤逆转(交换 HP / MP)|

### 2.4 特技数据结构

```csharp
public class SpecialSkillData {
    public int SkillId;
    public string Name;
    public int AngerCost;       // 愤怒消耗(常见 50-150)
    public TargetType TargetType;
    public int EffectId;        // 效果 id
    public string Formula;      // 伤害公式字符串
    public int CooldownRounds;  // CD 回合数
    public bool IsAoeBuff;      // 是否群体 buff
}

public class SpecialSkillInstance {
    public SpecialSkillData Definition;
    public int CurrentCd;       // 当前 CD 计数
}
```

### 2.5 特技 vs 普通技能

| 维度 | 特技 | 普通技能 |
|------|------|----------|
| 消耗 | 愤怒(50-150)| MP(常见 30-80)|
| 频率 | 低(每场战斗 1-3 次)| 高(每回合 1 次)|
| 效果 | 强(全体 / 大量) | 普通 |
| 决策 | 高(愤怒有限)| 低(MP 充足)|
| 学习 | 装备 / 任务 / 等级 | 升级自动学 |

### 2.6 特技的"释放流程"

```
1. 玩家选特技
2. 选目标(根据 TargetType)
3. 检查愤怒够不够
4. 愤怒不够 → 提示"愤怒不足"
5. 愤怒够 → 扣愤怒 + 进 CD
6. 结算效果(读公式)
7. 播动画 + 弹伤害数字
```

## 三、药品系统

### 3.1 药品的本质

药品 = "战斗内的消耗品":
- 战斗中用(临时符 / 金创药)
- 战斗中限制次数(常见 1-3 个药品栏)
- 战斗外用(任务物品 / 永久药)

### 3.2 药品的设计目标

1. **补充资源** —— 战斗中补 HP / MP / 解除异常。
2. **决策维度** —— "现在用药还是等下一回合"。
3. **战前策略** —— "带什么药进战斗"。

### 3.3 药品的常见分类

| 类别 | 描述 | 典型 |
|------|------|------|
| **HP 药** | 恢复 HP | 金创药(回 30% HP)|
| **MP 药** | 恢复 MP | 灵芝(回 50% MP)|
| **复活药** | 救起倒地 | 天复灵药(复活 + 满血)|
| **异常解除** | 净化 | 五龙丹(解除中毒 / 封印)|
| **属性增强** | 临时加属性 | 临时符(进战斗前加愤怒 / 加属性)|
| **战前准备** | 战斗前永久加属性 | 永久加属性符 |
| **复活类** | 救起队友 | 救死扶伤药 |

### 3.4 药品数据结构

```csharp
public class ItemData {
    public int ItemId;
    public string Name;
    public ItemType Type;        // HP / MP / 复活 / 异常解除 / 临时符 / ...
    public string EffectFormula; // 效果公式
    public int MaxStack;         // 最大叠加数
    public int UseLimit;         // 战斗中可用次数(0 = 无限)
}

public class ItemInstance {
    public ItemData Definition;
    public int Count;           // 当前数量
}
```

### 3.5 药品 vs 技能

| 维度 | 药品 | 技能 |
|------|------|------|
| 消耗 | 物品数量 | MP |
| 频率 | 限制次数(1-3)| 限制次数(CD)|
| 学习 | 物品可获得 | 升级 / 技能书 |
| 战斗前 | 选药品栏 | 选技能栏 |
| 战斗中 | 用物品按钮 | 选技能按钮 |

### 3.6 战斗内药品栏

- 战斗中药品栏固定 N 个(常见 3-6 个)
- 每个药品栏有"可用次数"(常见 1-3 次)
- 战斗结束 → 药品栏重置

### 3.7 药品的" +30% HP"是相对最大 HP

**重要规则**:药品的"+30% HP"是相对最大 HP,**不是**绝对值。

```csharp
// ✅ 正确:相对最大 HP
var maxHp = target.GetComponent<ResourceComponent>().GetMax(ResourceId.HP);
var heal = (long)(maxHp * 0.3);
target.GetComponent<ResourceComponent>().Modify(ResourceId.HP, heal);

// ❌ 错误:绝对值
target.GetComponent<ResourceComponent>().Modify(ResourceId.HP, 30);
```

**设计意图**:
- 高级玩家(高 HP)的"30% 药品"也回得多 → 药品价值随等级提升
- 平衡"低等级玩家 + 高级药品"的过度收益

## 四、愤怒 / 特技 / 药品的协同

### 4.1 三者关系

```
愤怒累积 → 释放特技 → 逆转战局(决策点)
道具使用 → 补 HP / MP → 维持战斗(决策点)
愤怒 + 道具 = 复活药(愤怒消耗 + 物品消耗)
```

### 4.2 战斗内决策树

```
每个回合:
    我需要做什么?
    ├─ 当前回合能秒对方 → 攻击(浪费愤怒)
    ├─ 当前回合可能被秒 → 防御 + 加血
    ├─ 愤怒满了(150) → 放特技
    ├─ 队友倒地 → 复活药
    ├─ 队友被封 → 五龙丹
    ├─ 队友缺 HP/MP → HP/MP 药
    └─ 默认 → 攻击 / 防御
```

### 4.3 三者的"价值曲线"

| 资源 | 累积速度 | 使用频率 | 价值 |
|------|---------|---------|------|
| **HP** | 持续消耗(被打) | 高(每回合)| 基础 |
| **MP** | 持续消耗(放技能)| 高(每回合)| 基础 |
| **愤怒** | 慢(N 回合累积)| 低(每场 1-3 次)| 高(翻盘点)|
| **药品** | 战斗前消耗(带几个)| 低(限制次数)| 高(关键回合)|
| **特技** | 慢(愤怒满)| 低(愤怒满)| 极高(逆转)|

## 五、愤怒 / 特技 / 药品系统的常见边界

### 5.1 愤怒 vs 资源

| 愤怒 | 资源(HP / MP)|
|------|--------------|
| 累积式(被打)| 消耗式(被打 / 放技能)|
| 上限固定(150) | 上限绑定基础属性 |
| `AngerComponent`(自定义)| `ResourceComponent`(内置)|

### 5.2 愤怒 vs 状态(state / Provider)

> 详见 [`05-state-component.md`](05-state-component.md) 和 [`06-buff-system.md`](06-buff-system.md)。

| 愤怒 | 状态(state) | buff Provider |
|------|-------------|--------------|
| 累积式(数值) | 在 / 不在(集合成员关系) | 业务对象,持有 duration |
| 持续整个战斗 | 持续 N 回合 | 自己持有 RemainingRounds |
| 跟资源系统关联 | 跟 StateComponent 关联 | 跟 StateComponent 协作 |

### 5.3 愤怒系统的常见反模式

| 反模式 | 问题 |
|--------|------|
| **愤怒无上限** | 后期愤怒累积太快 → 失去设计意图 |
| **愤怒太快** | 每回合愤怒满 → 特技变成"普通技能"|
| **愤怒太慢** | 战斗结束都愤怒不满 → 特技成摆设 |
| **愤怒无减怒** | 单方滚雪球(愤怒累积越快)→ 失衡 |
| **愤怒无敌化** | 愤怒满后无敌 → 玩家无对抗手段 |

## 六、本章沉淀的设计模式

沉淀的不是具体某款 RPG 的愤怒 / 特技 / 药品数值,**而是这些系统的设计骨架**:

### 愤怒系统:
1. **加怒分段式** —— 按损失气血比例分段,残血累积更快。
2. **死亡清 0** —— 倒地 / 死亡 / 救起都清 0。
3. **多段攻击分别计算** —— 横扫千军每刀算 1 次。
4. **主动消耗血不增怒** —— 防止"消耗血加愤怒"套利。
5. **愤怒腰带削 20%** —— 装备特化。

### 特技系统:
1. **消耗愤怒 + 进 CD** —— 释放成本高,不能频繁。
2. **群体 buff / debuff / 伤害 / 复活 / 治疗** —— 6 大类。
3. **效果强 + 决策点** —— 愤怒有限,何时放哪个。

### 药品系统:
1. **战斗中限制次数(1-3 个药品栏)** —— 决策维度。
2. **+30% HP 是相对最大 HP** —— 平衡高低等级。
3. **战斗前选药品栏** —— 战前策略。

### 三者协同:
1. **愤怒是翻盘点** —— 残血一波反杀。
2. **药品是维持点** —— 维持战斗节奏。
3. **特技是爆发点** —— 决定回合结果。

具体数值(愤怒上限 150、罗汉金钟消耗 150、金创药 +30% HP)是梦幻西游的数值,**不**沉淀到这里。
